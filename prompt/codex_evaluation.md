You are an evaluator for AI-generated educational Google Colab notebooks.

I will provide you with a Jupyter / Colab notebook generated by Claude Code. Your task is to evaluate whether the notebook is suitable as an interactive demo for a computer science or machine learning course.

Please evaluate the xxx notebook using the following criteria:

1. Executability
- Can the notebook run from top to bottom without errors?
- Are there missing dependencies, broken imports, undefined variables, or runtime issues?
- If the notebook uses Colab-specific features, are they handled properly?
- If GPU is used, does it correctly detect CUDA and fall back to CPU when unavailable?

2. Concept Correctness
- Does the notebook correctly explain and implement the target concept?
- Are there any algorithmic, mathematical, or machine learning mistakes?
- Are edge cases handled correctly?
- Does the code behavior match the explanation?

3. Interactivity
- Does the notebook include meaningful interactive elements?
- Are widgets, sliders, buttons, dropdowns, or editable inputs actually useful for learning?
- Do changes in parameters lead to visible and understandable changes in output?
- Is the interaction robust, or does it break easily?

4. Visualization Quality
- Are the visualizations clear and relevant?
- Do they help students understand the concept?
- Are labels, legends, titles, and axes understandable?
- For algorithm demos, does the visualization show important intermediate states?
- For ML demos, does it show things like loss curves, decision boundaries, predictions, or model behavior?

5. Pedagogical Value
- Does the notebook have clear learning objectives?
- Does it explain the concept in a beginner-friendly way?
- Does it guide students through observation, experimentation, and reflection?
- Does it include student exercises or reflection questions?
- Does it address common misconceptions?

6. Alignment with Source Material
- If source slides, transcript, or course material are provided, does the notebook stay aligned with them?
- Does it focus on the intended concept instead of adding unrelated content?
- Does it preserve the level and scope of the original course material?

7. Robustness
- Does the notebook still work if the user changes parameters?
- Are invalid inputs handled gracefully?
- Does the notebook avoid fragile hard-coded assumptions?
- Does it avoid relying on local files or paths that will not exist in Colab?

8. Simplicity and Maintainability
- Is the code readable and well-organized?
- Is the notebook unnecessarily complex?
- Are functions modular enough to understand?
- Are comments helpful without being excessive?
- Is the notebook appropriate for students rather than over-engineered?
